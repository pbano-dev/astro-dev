## Overview

Human Fall Flat 2 is the professional production project that best represents my C# software engineering experience. The portfolio angle here is broader than Unity or gameplay: it is about building maintainable runtime systems, internal tools and debug workflows that can survive iteration, review, QA pressure and milestone stabilization.

## My contribution

- Designed and implemented C# systems for runtime behaviour, tools, debugging and gameplay-facing interactions.
- Built modular code using assemblies, internal packages, reusable components, ScriptableObjects, Prefabs and Addressables.
- Translated ambiguous production and design requirements into implemented, reviewable and testable software features.
- Created internal tools and debug utilities used during iteration, QA investigation and bug fixing.
- Worked with proprietary multiplayer technology, physics-driven features, event logic and FMOD gameplay integration.
- Participated in Git pull requests, code reviews, Jira workflows and cross-functional discussions with programmers, designers, QA, audio and production.

## Technical focus

The key story is maintainability under production constraints. I worked on code that needed to be readable, reviewable, debuggable and safe to iterate on across disciplines. That included architectural decisions, feature-level ownership, debug tooling, bug reproduction, profiling-driven fixes and memory / GC optimization where appropriate, while respecting NDA boundaries around unreleased production details.
