## Overview

SufferEngine is the strongest C++ systems project in the portfolio. It was not a simple rendering demo: it combined runtime architecture, an entity/component model, transform hierarchy, scripting, audio execution, resource management, editor tooling and rendering into a coherent custom engine.

## My contribution

- Implemented systems-level functionality across transforms, parent/child hierarchy and component lifecycle.
- Worked on resource loading, custom memory structures, allocators / pools and runtime/editor integration.
- Built editor tools with Dear ImGui, including hierarchy, inspector, component editing, shader controls, lighting toggles and audio playback controls.
- Contributed to OpenGL rendering, specifically normal mapping and lighting implementation.
- Integrated third-party libraries such as GLFW, GLEW, Dear ImGui, px_sched, stb_image, SoLoud and tinyobjloader while maintaining a custom architecture.

## Technical focus

This case study shows lower-level engineering beyond commercial engine usage: architecture, data ownership, component lifecycle, editor/runtime boundaries, resource management and rendering integration. It is the clearest counterweight to the assumption that my work is limited to Unity or gameplay scripting.
